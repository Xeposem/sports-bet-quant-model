    **Kelly module** — Create `src/backtest/kelly.py` with:

    - `compute_kelly_bet(prob, decimal_odds, bankroll, kelly_fraction=0.25, max_fraction=0.03, min_ev=0.0) -> float`:
      - EV = (prob * decimal_odds) - 1. If EV < min_ev, return 0.0.
      - Full Kelly = (b*p - q) / b where b = decimal_odds - 1, p = prob, q = 1-p.
      - If full_kelly <= 0, return 0.0 (negative Kelly = skip bet, logged as "passed").
      - Fractional = full_kelly * kelly_fraction.
      - Capped = min(fractional * bankroll, max_fraction * bankroll).
      - Return capped.
    - `apply_bet_result(bankroll, bet_size, decimal_odds, won) -> float`:
      - If won: bankroll + bet_size * (decimal_odds - 1).
      - If lost: bankroll - bet_size.

    Create `src/backtest/walk_forward.py` with these functions:

    - `generate_folds(conn, min_train_matches=500) -> list[tuple[str, str, str]]`:
      - Query distinct years from matches table (where match_features exist).
      - For each year: train_end = "{year}-01-01", test_start = "{year}-01-01", test_end = "{year+1}-01-01".
      - Count training rows: `SELECT COUNT(*) FROM match_features mf JOIN matches m USING (tourney_id, match_num, tour) WHERE m.tourney_date < :train_end AND mf.player_role = 'winner'`.
      - Skip folds with insufficient training data. Return list of (train_end, test_start, test_end) tuples.

    - `build_fold_training_matrix(conn, train_end: str) -> tuple[np.ndarray, np.ndarray, list[str]]`:
      - Copy the `_BUILD_MATRIX_SQL` pattern from trainer.py but add `AND m.tourney_date < :train_end` before `ORDER BY`.
      - Returns (X, y, match_dates) same format as `build_training_matrix`.
      - CRITICAL: Must use the exact same differential column expressions as trainer.py `_BUILD_MATRIX_SQL`.

    - `build_fold_test_matches(conn, test_start: str, test_end: str) -> list[dict]`:
      - Query all matches in test window that have both match_features AND match_odds.
      - Each dict contains: tourney_id, match_num, tour, winner_id, loser_id, surface, tourney_level, tourney_date, decimal_odds_a, decimal_odds_b, winner_rank (from match_features), loser_rank.
      - Also query matches WITH features but WITHOUT odds — log them as "no_odds" (skip bet).

    - `assert_no_look_ahead(train_dates: list[str], test_dates: list[str]) -> None`:
      - Asserts no date in test_dates appears in train_dates. Raises AssertionError with details if violated.

    - `run_fold(conn, train_end, test_start, test_end, bankroll, config) -> tuple[list[dict], float]`:
      - 1. Build fold training matrix via build_fold_training_matrix(conn, train_end).
      - 2. Compute time weights with reference_date=train_end.
      - 3. temporal_split 80/20 within training data.
      - 4. train_and_calibrate to get calibrated model.
      - 5. For each test match with odds: compute calibrated prob via model.predict_proba on the differential feature vector (same pattern as _FEATURE_QUERY in predictor.py but for test window). Compute EV via compute_ev. Compute Kelly bet via compute_kelly_bet.
      - 6. If bet > 0: determine outcome (winner_id == player we bet on?), compute pnl, update bankroll.
      - 7. Build backtest_results row for each bet decision (including passed/no_odds).
      - 8. Return (results_rows, updated_bankroll).

    - `run_walk_forward(conn, config=None) -> dict`:
      - config defaults: kelly_fraction=0.25, max_fraction=0.03, min_ev=0.0, initial_bankroll=1000.0, min_train_matches=500, model_version="logistic_v1".
      - Calls generate_folds, then iterates run_fold for each, carrying bankroll forward.
      - INSERT OR REPLACE all results into backtest_results table.
      - Returns summary: {folds_run, total_bets, bets_placed, bets_skipped, total_pnl_kelly, total_pnl_flat, final_bankroll, start_bankroll}.

    **Key implementation notes:**
    - The test-set prediction must use the SAME pairwise differential convention as training: query winner+loser rows from match_features, compute feature vector from winner-perspective, model predicts P(winner wins).
    - For betting: we always bet on the player with positive EV. If winner has EV > 0, bet on winner (outcome=1 if winner won, which is always true in historical data since winner_id IS the winner). If loser has EV > 0, bet on loser (outcome=1 if loser won, which is always 0 in historical data). Both sides can have positive EV — compute both.
    - Matches without odds: log row with kelly_bet=0, pnl=0, outcome still recorded.
    - Flat bet is always $1 for comparison ROI calculation.
